## Состав пакета

| Файл | Назначение | Куда ставится |
|---|---|---|
| `flux.lua` | Основной мониторинг: 3 вкладки (Энергия / ЦПУ / Сеть ME) | `/home/flux.lua` |
| `renderer.lua` | Графическая библиотека: палитра, прогресс-бары, спарклайны, шрифты | `/lib/renderer.lua` |
| `viewer.lua` | Текстовый просмотрщик с прокруткой (OC терминал листать не умеет) | `/home/viewer.lua` |
| `component_probe.lua` | Сканер всех OC-компонентов в сети с эвристикой назначения | `/home/component_probe.lua` |
| `install.lua` | Установщик: скачивает файлы из этого репо и настраивает автозапуск | временно в `/tmp/` |

## Возможности

### Главный мониторинг (`flux.lua`)
- **3 вкладки** с переключением кликом по корешку: Энергия / ЦПУ / Сеть ME
- **Спарклайны истории** входа/выхода/заполнения за последние 60 опросов
- **Прогноз ETA** до полного или пустого буфера (тренд-стрелки `▲▼`)
- **Звуковые тревоги** при низком заряде или дефиците (настраивается)
- **Автокалибровка максимума** — если в сети накопится больше чем считалось максимумом, значение автоматически расширяется до фактического пика
- **Сохранение настроек** между перезапусками в `/etc/fluxmon.cfg`
- **Управление полностью на тач-экране** — никаких сочетаний клавиш

### Графика (`renderer.lua`)
- Кастомная киберпанк-палитра 16 цветов через `gpu.setPaletteColor`
- Субсимвольный прогресс-бар с точностью до 1/8 ячейки
- Двухстрочные спарклайны на 17 уровней высоты
- Шрифт 3×6 на полублоках для крупных цифр
- Анимированная подпись с бегущим бликом

### Утилиты
- `viewer.lua` — текстовый просмотрщик с прокруткой колесом мыши и скроллбаром, подсветка для логов (`✓`, `✗`, `===`, `>>>`, `ERROR`)
- `component_probe.lua` — сканер ВСЕХ компонентов сети с трёхуровневой эвристикой назначения (имя → подстроки → сигнатура методов). Распознаёт компоненты HiTech: Flux, AE2, Computronics, Energy Control, OpenSecurity, OpenGlasses, Modular Machinery, Draconic, Ender IO, IC2, Forestry, OpenExtensions

## Установка в OpenComputers

### Шаг 1. Подключение к интернету
В компьютер OC должна быть вставлена **Internet Card Tier 2**. Проверка:
```
lua =component.internet
```
Если выдаёт таблицу — всё работает. Если `nil` — карта не вставлена или Tier 1.

### Шаг 2. Скачивание установщика
```
wget -f https://raw.githubusercontent.com/ShadowStormOne/flux-monitor/main/install.lua /tmp/i.lua
```

### Шаг 3. Запуск установки
```
/tmp/i.lua
```

Скрипт:
1. Скачает все файлы пакета из репозитория
2. Положит их в правильные директории (`/lib/`, `/home/`)
3. Спросит про автозапуск через `rc.d`

### Шаг 4. Запуск
```
flux                       # основной мониторинг
viewer /home/file.log      # просмотрщик файлов
component_probe            # сканер всех компонентов
```

## Управление автозапуском

После установки `flux` запускается автоматически при включении ПК как сервис `flux_monitor` через стандартный rc-демон OpenOS.

```
rc flux_monitor start      # запустить вручную
rc flux_monitor stop       # остановить
rc flux_monitor restart    # перезапустить (после правок кода)
rc flux_monitor status     # узнать состояние
rc flux_monitor enable     # включить автозапуск
rc flux_monitor disable    # выключить автозапуск
```

Полностью удалить из автозапуска:
```
/tmp/i.lua --no-autostart
```

## Конфигурация

Все настройки хранятся в `/etc/fluxmon.cfg`:

```lua
{
  useCompactFormat = true,    -- "1.41 GRF" вместо "1.408.000.000 RF"
  maxTotalEnergy   = 13e9,    -- макс. ёмкость буфера, авто-калибруется
  autoMaxEnergy    = true,    -- расширять максимум при превышении
  alarmLowPct      = 25,      -- порог тревоги по низкому заряду (%)
  alarmDeficitSec  = 5,       -- сек. дефицита перед сиреной
  soundEnabled     = true,    -- включены ли тревоги
  activeTab        = 1,       -- активная вкладка (1=Энергия, 2=ЦПУ, 3=ME)
  historyLength    = 60,      -- глубина истории спарклайнов
  scale            = 6.0,     -- масштаб экрана (для Renderer.init)
}
```

Редактировать можно прямо в OC через `edit /etc/fluxmon.cfg` либо удалить — пересоздастся с дефолтами.

## Обновление существующей установки

Если уже установлено и хочешь подтянуть свежую версию из репо:

```
wget -f https://raw.githubusercontent.com/ShadowStormOne/flux-monitor/main/install.lua /tmp/i.lua
/tmp/i.lua -f
```

Флаг `-f` означает «перезаписать существующие файлы». Без него установщик пропустит уже существующие.

После обновления перезапуск сервиса:
```
rc flux_monitor restart
```

## Для разработчика — обновление кода на GitHub

> **Это раздел для меня самого, чтобы не забыть workflow.**

Все файлы лежат в `C:\Users\Admin\Desktop\git`.

### Стандартный workflow после правок

```powershell
cd C:\Users\Admin\Desktop\git

# Посмотреть какие файлы изменились
git status

# Посмотреть конкретные изменения (опционально)
git diff

# Закоммитить ВСЕ изменения
git add .
git commit -m "Краткое описание изменений"

# Отправить на GitHub
git push
```

После этого изменения видны на `https://github.com/ShadowStormOne/flux-monitor`, а в OC можно подтянуть командой `/tmp/i.lua -f`.

### Если только что-то одно поменял

```powershell
git add flux.lua
git commit -m "Fix: trend arrow color when net=0"
git push
```

### Откатить незакоммиченные правки

Если что-то сломал и хочешь вернуть как было на гитхабе:

```powershell
git checkout -- flux.lua          # вернуть один файл
git checkout -- .                 # вернуть все изменения
```

### Посмотреть историю коммитов

```powershell
git log --oneline -10             # последние 10 коммитов
git log --stat                    # с указанием изменённых файлов
```

### Если push отклоняется (`rejected`)

Кто-то/что-то закоммитило в репо мимо локалки (например правка через веб-интерфейс GitHub). Сначала забрать удалённые изменения, потом запушить:

```powershell
git pull --rebase
git push
```

### Создание нового файла

Просто кладёшь его в `C:\Users\Admin\Desktop\git`, потом обычный workflow:

```powershell
git add .
git commit -m "Add: new_script.lua"
git push
```

⚠️ **Не забудь добавить новый файл в массив `FILES` внутри `install.lua`!** Иначе установщик его не увидит и в OC он не попадёт.

### Если забыл что куда коммитить

```powershell
git status                        # Git подскажет что делать
```

Git очень разговорчивый — читай вывод `git status`, там всегда написано какую команду выполнить дальше.
